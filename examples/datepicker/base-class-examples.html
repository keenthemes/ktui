Datepicker Base Class + Overrideable Class Examples

1. Base Classes Only

Using only the default base classes without any custom overrides. Base classes provide consistent default styling.

CSS Used: 
.kt-datepicker-header {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  color: white;
  padding: 1rem;
  border-radius: 0.5rem 0.5rem 0 0;
}

.kt-datepicker-day-cell {
  border-radius: 0.25rem;
  transition: all 0.2s ease;
}

2. Base + Overrideable Classes

Combining base classes with custom overrideable classes for enhanced styling.

Classes Applied:
  • kt-datepicker-header custom-header-override
  • kt-datepicker-day-cell custom-day-override
  • kt-datepicker-calendar-button custom-button-override

3. Tailwind CSS Overrides

Using Tailwind utility classes as overrideable classes for rapid styling.

Tailwind Classes:
  • bg-gradient-to-r from-pink-500 to-purple-600
  • hover:bg-pink-100 transition-colors
  • shadow-xl border-2 border-pink-200

4. Responsive Design

Classes that adapt to different screen sizes using responsive utilities.

Responsive Classes:
  • w-full md:w-80 - Full width on mobile, fixed width on desktop
  • text-xs md:text-sm - Smaller text on mobile
  • p-2 md:p-3 - Less padding on mobile

5. Dark Theme

Custom dark theme styling using both base and overrideable classes.

Dark Theme Classes:
  • dark-theme - Dark background and text
  • text-white - White text for dark backgrounds
  • bg-gray-800 - Dark input background

6. Custom Template + Classes

Combining custom templates with base and overrideable classes.

Template + Classes:
  • kt-datepicker-day-cell custom-day-cell border border-gray-200
  • kt-datepicker-header custom-header-wrapper bg-gradient-to-r from-indigo-500 to-purple-600

Understanding Base Class + Overrideable Class Pattern

Class Structure

Each template element uses the pattern: class="kt-datepicker-{element} {{class}}"

Base Classes

Base classes (e.g., kt-datepicker-header) are always present and provide:

  • Consistent default styling across all instances
  • Semantic class names for targeting
  • Foundation for component theming

Overrideable Classes

Overrideable classes (via {{class}}) allow:

  • Per-instance customization
  • Integration with CSS frameworks
  • Dynamic styling based on state

Best Practices

  • Always include base classes in custom templates
  • Use semantic class names for overrideable classes
  • Test responsive behavior with different screen sizes
  • Maintain accessibility when customizing styles

CSS Targeting Examples

/* Target all datepicker headers */
.kt-datepicker-header {
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
}

/* Target specific instance with custom class */
.kt-datepicker-header.custom-theme {
  background: linear-gradient(135deg, #ff6b6b 0%, #ee5a24 100%);
}

/* Target base class with Tailwind utilities */
.kt-datepicker-day-cell.hover\:bg-blue-100:hover {
  background-color: #dbeafe;
}
