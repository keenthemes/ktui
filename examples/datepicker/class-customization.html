How to Use Class Customization

1. Basic Usage

Add a classes object to your datepicker config:

data-kt-datepicker-config='{
  "classes": {
    "header": "your-header-class",
    "dayCell": "your-day-cell-class",
    "dropdown": "your-dropdown-class"
  }
}'

2. Available Class Properties

You can customize classes for any template element:

  • container - Main container
  • header - Calendar header
  • footer - Footer actions
  • calendarGrid - Calendar table
  • dayCell - Individual day cells
  • calendarButton - Calendar button
  • dropdown - Dropdown container
  • And many more...

3. Combining with Custom Templates

You can use class placeholders in custom templates:

"dayCell": "<td class=\"custom-base {{class}}\">{{day}}</td>"

4. Best Practices

  • Use semantic class names that describe the purpose
  • Combine with CSS frameworks like Tailwind for rapid styling
  • Test responsive behavior with different screen sizes
  • Ensure accessibility by maintaining proper contrast ratios
